基于GitHub Pages的博客搭建实验报告

返回主页

实验目的

掌握基于GitHub云平台的静态网站部署技能，实现个人博客/网站的搭建与自动化发布，完成实验报告的在线展示。

实验环境

硬件环境

  • 联网计算机（Windows/Mac/Linux）

软件环境

  • Git 2.40+
  • VS Code编辑器
  • Chrome浏览器

实验步骤

  1. 创建GitHub仓库

    新建名为 username.github.io 的公开仓库（需与账号同名）

  2. 本地开发环境搭建

    # 克隆仓库
git clone https://github.com/username/username.github.io
cd username.github.io

# 基础文件结构
├── index.html
├── experiment.html
├── css/
│   └── style.css
└── images/

  3. 页面开发

    使用HTML5语义化标签构建：

    • 主页包含个人介绍与实验报告入口
    • 实验报告页呈现完整实验文档

  4. 自动化部署

    通过Git提交触发GitHub Pages构建：

    git add .
git commit -m "完成实验报告页面"
git push origin main

  5. 访问验证

    在浏览器访问 https://username.github.io 验证部署结果

实验结论

成果展示

  • 成功通过GitHub Pages实现静态网站托管
  • 完成包含双向导航的响应式网站架构
  • 实验报告访问量统计（通过GitHub Insights）

技术要点

  • Git版本控制在云部署中的应用
  • GitHub Actions自动化构建流程
  • 基于Issues的博客评论系统集成
